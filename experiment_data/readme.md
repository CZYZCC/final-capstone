# SmartQG Experimental Evaluation Dataset (1,800 Questions)

This dataset contains the experimental results generated by the SmartQG pipeline. It comprises 1,800 highly structured, psychometrically evaluated Computer Science exam questions across six model configurations.

## How to Read the Dataset (Data Schema)

The dataset is provided in JSON Lines (`.jsonl`) format. Each line represents a single generated question combined with its metadata and automated psychometric evaluation. 

A single JSON record contains three primary top-level keys: `metadata`, `question`, and `score`.

### 1. `metadata` (Generation Context)
This object tracks the exact configuration and execution context of the generated question.
* `topic`: The specific sub-topic (e.g., "merge sort", "cybersecurity").
* `model`: The model configuration used (e.g., `graph_rag`, `hybrid_rag`, `no_retrieval`).
* `method`: The specific generation path taken (e.g., `generated`, `generated_difficulty_retry_x2` indicating it triggered the self-correction loop).
* `question_format`: The format of the question (`mcq_single`, `mcq_multi`, `true_false`, `fill_blank`, `open_answer`).
* `generation_time`: Time taken to generate the question in seconds.

### 2. `question` (Generated Content & Cognitive Tracing)
This object contains the actual question payload, the retrieved context, and the LLM's internal cognitive trace.
* `question` / `statement` / `sentence`: The main prompt presented to the student (key varies by format).
* `correct_answer` / `answers`: The mathematically or logically verified correct answer.
* `rationale`: The step-by-step reasoning proving the correct answer.
* `distractors`: (For MCQ formats) An array of incorrect options. **Crucially, each distractor's `explanation` begins with a specific Error Tag** (e.g., `[Procedural Omission]`, `[Operator Confusion]`, `[Boundary Error]`), mapping the wrong answer to a specific student misconception.
* `retrieved_context`: The raw knowledge graph subgraph or vector chunks injected into the prompt before generation.
* `generator_scratchpad`: The LLM's internal step-by-step mathematical trace and justification for its adversarial edge cases, generated *before* finalizing the question text.

### 3. `score` (Psychometric Evaluation)
This object contains the output of the `AutomatedEvaluator` judge, which strictly scores the question on a 1.0 - 5.0 scale across 7 dimensions.
* `overall`: The weighted aggregate score.
* `relevance` (5%): Alignment with the target topic.
* `correctness` (20%): Mathematical and logical flawlessness.
* `diagnostic_power` (20%): How well the distractors isolate specific cognitive errors.
* `multi_hop_dependency` (15%): The depth of cross-concept reasoning required to solve it.
* `edge_case_triggering` (20%): The presence of adversarial inputs or boundary conditions.
* `graph_relational_depth` (10%): The degree to which the question synthesizes the provided graph topology.
* `diversity` (10%): Session-level format and topic novelty.
* `details`: Contains the raw LLM judge output, including a `verdict` (e.g., "CORRECT") and a detailed textual `critique_feedback` explaining the exact deductions.

```json
{
  "metadata": {
    "topic": "merge sort",
    "model": "graph_rag",
    "method": "generated",
    "question_format": "mcq_single",
    "expected_type": "computational"
  },
  "question": {
    "question": "Consider implementing two sorting algorithms on the already-sorted array [1, 2, 3, 4, 5, 6, 7, 8]... How many MORE comparisons does Algorithm A make than Algorithm B?",
    "correct_answer": "11",
    "rationale": "For quicksort... worst-case O(n²) behavior. Total = 28. For merge sort... total = 17. The difference is 28 - 17 = 11.",
    "distractors": [
      {
        "option": "28",
        "explanation": "[Procedural Omission] Student correctly computed quicksort's 28 comparisons but forgot to subtract merge sort's 17 comparisons..."
      }
    ]
  },
  "score": {
    "overall": 4.55,
    "correctness": 5.0,
    "diagnostic_power": 5.0,
    "multi_hop_dependency": 4.5,
    "edge_case_triggering": 4.0,
    "graph_relational_depth": 1.0,
    "details": {
      "verdict": "CORRECT",
      "critique_feedback": "The question is well-constructed with strong diagnostic power through specific key points. It demonstrates excellent multi-hop reasoning by requiring a causal chain..."
    }
  }
